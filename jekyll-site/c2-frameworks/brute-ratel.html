Brute Ratel C4 (BRC4)

Brute Ratel C4 is a commercial red team C2 framework by Chetan Nayak (@NtRaiseHardError). Its "Badger" agent is designed with EDR evasion as the primary goal — no Cobalt Strike-derived code, unique syscall implementation, and direct kernel object manipulation. BRC4 gained notoriety when leaked in 2022; licensed versions are available for legitimate red teams.

Architecture Overview

# BRC4 Components:
# - Server: C2 backend (Windows or Linux via Wine)
# - Client (BRc4.exe): operator GUI
# - Badger: agent (x64/x86, EXE/DLL/shellcode)
# - Profiles: HTTP/S/DNS/SMB/TCP communication configurations

# Badger evasion features:
# - API hashing (no import table of sensitive functions)
# - Stack encryption during sleep
# - Indirect syscalls via custom Hell's Gate implementation
# - Thread call stack spoofing
# - ETW/AMSI patching
# - Process injection without NtWriteVirtualMemory hooks
# - Module stomping for shellcode allocation
# - Parent process ID spoofing
# - Block non-Microsoft DLLs (PROCESS_CREATION_MITIGATION_POLICY)

Unique Evasion Techniques

# BRC4 differentiators:

# 1. No suspicious imports in Badger PE:
#    All NTAPI resolved via API hashing
#    Import table only shows innocuous functions
#    PEsieve, Hunt-Sleeping-Beacons, Moneta won't find Cobalt Strike IOCs

# 2. SleepMask (proprietary):
#    Encrypts entire Badger memory + PE headers during sleep
#    Uses ROP chain to call encryption routines from ntdll
#    Stack shows NtWaitForSingleObject called from RtlUserThread

# 3. Process Injection (custom):
#    Doesn't use classic WriteProcessMemory + CreateRemoteThread
#    Uses NtMapViewOfSection-based injection:
#    Create section → NtMapViewOfSection into target → APC to execute
#    No NtWriteVirtualMemory calls (commonly monitored)

# 4. UDRL (User Defined Reflective Loader):
#    Custom reflective DLL loader (not the classic Stephen Fewer version)
#    Hides PE headers after loading
#    Randomizes sleep timer per-agent using hardware RDTSC

# 5. DNS C2:
#    DNS over HTTPS (DoH) profile available
#    Data encoded in TXT/A/AAAA DNS records
#    Can bypass strict egress filtering

Detection & OPSEC Notes

# BRC4 IOCs post-2022 leak:

# Network:
# - Hardcoded User-Agent strings from leaked profiles
# - Specific URI patterns in leaked configs
# - Certificate fingerprints from known C2 infra

# Memory:
# - Leaked Badger DLL signatures in AV databases (post-2022)
# - Custom legitimate versions should be compiled from source
# - Badger UDRL doesn't fully hide PE headers in older versions

# OPSEC best practices:
# - Generate new Badger builds per engagement
# - Use CDN redirectors (Cloudflare, Fastly) as front
# - Custom profiles with engagement-specific URIs
# - Rotate SSL certificates per engagement
# - Use DNS profile for air-gapped / restricted environments

# Hunting BRC4:
# Look for processes with:
# - PE but no MZ header at base (header stomping)
# - Executable memory not backed by file on disk
# - RWX memory regions without corresponding module
